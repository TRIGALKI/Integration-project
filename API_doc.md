Документация API: Сервис предсказания дефолта

Этот документ описывает API веб-сервиса, предназначенного для предсказания вероятности дефолта по кредиту.

1. Общая информация
Базовый URL: http://<адрес_сервера>:5000/ (например, http://localhost:5000/)
Формат данных: JSON
Кодировка: UTF-8
2. Доступные эндпоинты
2.1. Проверка работоспособности сервиса: /health

Этот эндпоинт используется для проверки, что сервис запущен и модели успешно загружены.

Метод: GET
URL: http://<адрес_сервера>:5000/health

Ответ:

Успех (200 OK): Сервис работает, и хотя бы одна модель загружена.
```json
{
  "status": "healthy",
  "loaded_models": ["1", "2"]
}
```
Ошибка (503 Service Unavailable): Сервис работает, но ни одна из моделей не была загружена.
```json
{
  "status": "unhealthy",
  "error": "Модели не загружены"
}
```
2.2. Получение предсказания: /predict

Основной эндпоинт для получения предсказания вероятности дефолта.

Метод: POST
URL: http://<адрес_сервера>:5000/predict
Заголовок: Content-Type: application/json

Тело запроса (Request Body):

Тело запроса должно содержать JSON-объект с признаками клиента и, опционально, указанием версии модели.

Пример тела запроса:

```json
{
  "model_version": "2",
  "LIMIT_BAL": 20000,
  "SEX": 2,
  "EDUCATION": 2,
  "MARRIAGE": 1,
  "AGE": 35,
  "PAY_0": -1,
  "PAY_2": -1,
  "PAY_3": -1,
  "PAY_4": -1,
  "PAY_5": -1,
  "PAY_6": -1,
  "BILL_AMT1": 10000,
  "BILL_AMT2": 15000,
  "BILL_AMT3": 12000,
  "BILL_AMT4": 14000,
  "BILL_AMT5": 13000,
  "BILL_AMT6": 11000,
  "PAY_AMT1": 5000,
  "PAY_AMT2": 6000,
  "PAY_AMT3": 5500,
  "PAY_AMT4": 6500,
  "PAY_AMT5": 6200,
  "PAY_AMT6": 5800
}
```

Поля:

model_version (строка, опционально): Версия модели, которую следует использовать для предсказания. Допустимые значения — "1", "2". Если поле не указано, будет использована модель по умолчанию ("1").
<признак> (число): Все остальные поля в запросе являются признаками, на которых обучалась модель.
Их имена и порядок должны соответствовать данным, на которых обучалась модель.

Ответ:

Успех (200 OK):

```json
{
  "prediction": 1,
  "probability": 0.85,
  "model_version": "v2"
}
```

Ошибка клиента (400 Bad Request): Возникает при некорректном запросе 
(например, неверный формат JSON, запрошена несуществующая версия модели).

```json
{
  "error": "Модель версии v3 не найдена. Доступные версии: ['1', '2']."
}
```
Внутренняя ошибка сервера (500 Internal Server Error): Возникает при непредвиденной ошибке на стороне сервера во время обработки запроса.

```json
{
  "error": "Внутренняя ошибка сервера"
}
```
