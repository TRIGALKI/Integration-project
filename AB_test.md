# Организация A/B-тестирования (с учетом метрики Джини)

## Концепция A/B-теста

A/B-тестирование — это метод сравнения двух версий модели путем проведения эксперимента, в котором трафик пользователей случайным образом распределяется между этими версиями. Цель — определить, какая из версий лучше с точки зрения метрики Джини, и принять решение на основе данных.

В нашем сценарии мы сравниваем:

Контрольная группа (A): Текущая модель в продакшене (v1).
Тестовая группа (B): Новая, потенциально более качественная модель (v2).
Постановка A/B-теста

### 1. Способ разделения трафика:
Трафик от пользователей (заявок на кредит) должен распределяться случайным образом, чтобы обеспечить статистическую достоверность.

Распределение: 50% трафика направляется на модель v1, 50% — на модель v2.
Реализация: Для каждого входящего запроса генерируется случайное число. Если оно меньше 0.5, запрос обрабатывается моделью v1, иначе — v2. Для консистентности рекомендуется использовать user_id или request_id для закрепления пользователя за одной из групп на время теста.

### 2. Продолжительность теста:
Длительность теста определяется достижением статистической значимости, а не фиксированным периодом.

Тест должен продолжаться до тех пор, пока не будет набрано достаточное количество событий (фактов дефолта), чтобы сравнение метрик было достоверным. Это может занять от 1 до 4 недель в зависимости от объема трафика и базовой ставки дефолта. Необходимо заранее рассчитать требуемый размер выборки.
## Метрики и анализ
### 1. Бизнес-метрика для оценки
Основная и единственная метрика: Коэффициент Джини (Gini Coefficient)
Обоснование: Метрика Джини (нормализованная площадь под ROC-кривой) является стандартом де-факто в банковском секторе для оценки качества моделей кредитного скоринга. Она оценивает способность модели в целом отличать "хороших" заемщиков от "плохих", не завися от конкретного порога отсечения. Это делает ее идеальной бизнес-метрикой для сравнения общей предсказательной силы двух моделей.
### 2. Метод статистического анализа

Для сравнения метрики Джини двух независимых групп (A и B) используется двухвыборочный z-тест для пропорций или его непараметрические аналоги, адаптированные для ROC-анализа.

Как будут строиться доверительные интервалы:
Для каждой группы (A и B) будет рассчитан 95% доверительный интервал для значения метрики Джини. Разница между значениями Джини для v2 и v1 также будет представлена с доверительным интервалом.

Критерий успешности теста:
Тест считается успешным для новой модели (v2), если выполняются следующие условия:

Статистическая значимость: p-value теста меньше заранее выбранного порога (например, α = 0.05). Это означает, что наблюдаемая разница в метрике Джини между v1 и v2 с высокой вероятностью не является случайной.
Положительный эффект: Значение метрики Джини у тестовой группы (v2) статистически значимо выше, чем у контрольной (v1).

Если эти условия выполнены, принимается решение о выкатке новой модели (v2) на 100% трафика.
